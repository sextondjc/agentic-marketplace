# xUnit v2 to v3 Migration Checklist

## Baseline

- Current xUnit packages and versions inventoried.
- Runner adapter and platform dependencies identified.
- CI and local execution baselines captured.

## Compatibility

- Target runtime/SDK compatibility validated.
- Package compatibility blockers documented.
- Tooling and IDE integration checks completed.

## Phase Gates

- Phase 1 scope and success criteria defined.
- Phase 2 scope and success criteria defined.
- Final cutover success criteria defined.

## Rollback

- Rollback owner identified.
- Rollback commands and package versions documented.
- Rollback trigger conditions explicitly defined.

## Evidence

- Migration gate report artifact path defined.
- Residual risks and owners documented.
- Final recommendation: `proceed` | `pause` | `rollback`
